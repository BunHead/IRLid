## What we learned

- **The "endpoint live but user can't feel it" failure mode.** This is a class. Server-side correctness is necessary but not sufficient when the client's normal flow doesn't include a question whose answer is "your session is gone." Polling is unglamorous. Polling works. The session-poll heartbeat is 30 lines that closes a visibility gap a more elegant architecture (enforce Bearer everywhere) could close at the cost of much wider blast radius. The cheap correct thing was the right thing.

- **`git pull` before deploy is the new discipline rule.** Sounds obvious. Was not obvious in the moment when the curl-test loop assumed the local file was authoritative. Receipt: 30+ minutes spent diagnosing "why doesn't the new endpoint show up after Deploy" before searching for the new string in the deployed source and finding zero matches. The local file system is NOT necessarily a mirror of origin — pull first, then deploy. Promoted to BOOTSTRAP §6.

- **Cloudflare dashboard Quick Edit is the working fallback when wrangler API times out.** From Captain's home network this is a recurring pattern, not a one-off. The dashboard reaches Cloudflare via the same TLS handshake but a different routing path that succeeds where the management API doesn't. Worth using as the default for tactical deploys when wrangler hesitates. Promoted to BOOTSTRAP §6 with diagnostic recipe.

- **Cloudflare account selection during `wrangler login`.** Wrangler's browser flow defaults to whichever Cloudflare cookie session is active in the user's default browser — easy to authenticate into the wrong account silently. Verify via `wrangler whoami` and `wrangler d1 list` before any deploy. If d1-list shows zero databases on an account that should have at least one, you're in the wrong account.

- **The `Get-Clipboard | Measure-Object -Line` gotcha.** `Get-Content -Raw` reads a file as a SINGLE string object. `Measure-Object -Line` counts string objects, not embedded newlines, so it returns 1 even for a 3000-line file. Trust the editor's visible line count, not the PowerShell measure. The clipboard had the full content the whole time; the measurement was a red herring.

- **Mr. Data's autonomy on rebases is earning auto-merge tier.** PR #28 (v5.10.6 same-device sign-in) needed a rebase after v5.10.5 merged. Mr. Data noticed without prompting, did the rebase, force-pushed, came back with a clean diff. The kind of self-discipline that means his branches can be merged faster with less ceremony.
