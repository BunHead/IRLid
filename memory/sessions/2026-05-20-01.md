## Three patches worth banking specifically

**T4.3.31 — Stream rebuilt as JS particles.** The mockup needed real DOM particles to look like fire. CSS-only "random" doesn't exist (gradients are deterministic). The pattern: append N `<div>` elements at fire time with CSS custom properties driving per-particle position/colour/duration; animate via shared keyframe; self-clean via setTimeout. Each particle has random size (skewed small via `Math.pow(roll, 1.8)` so most are small with a few bigger ones — fire looks right that way), random colour from an 8-stop hot palette, random angle within a cone, random distance, random duration. The randomisation is what reads as "alive."

**T4.3.32 — Slider setting type added to the schema renderer.** Earlier T4.3.x patches added chips (`options: [...]` arrays) and the dir9 grid. T4.3.32 introduced sliders as a third UI type — `{type: 'slider', min: X, max: Y, step: Z, default: V}`. Updated `v511RenderSettingsForItem` with an `else if` branch for `type === 'slider'` that builds an `<input type="range">` + live value label. Added an `input` event listener on `v511CelSettingsBody` that updates `item.settings[key]`, refreshes the label, fires `v511SaveDebounced()`, and (T4.3.34) calls `v511UpdateAnchorCrosshairs()`. Now any future numeric setting can be a slider just by declaring it in the schema. Captain used this for particle count, stream width, anchor offset X/Y — and it'll be the right answer for any future "tune this number" UI.

**T4.3.35 — `mix-blend-mode: difference` for auto-contrast.** The crosshair was yellow `#ffd84a` with a box-shadow halo. On orange/yellow palettes it disappeared. Switched to white ink + `mix-blend-mode: difference` and the marker now inverts whatever pixel is underneath it — white on black, black on white, complementary on coloured. Always contrasts, no matter what's behind. This is the right answer for any UI marker that must remain visible against arbitrary backdrops — cursors, selection overlays, crosshairs, ruler guides. Worth banking as a pattern.
