### T4.3.1 — Sequence playback hotfix

Captain reported "only the 1st loaded effect when added to the sequence" plays. Diagnosed: when an effect class was removed and the next added in the same JavaScript tick (synchronously inside the `setTimeout` callback), the browser coalesced the two mutations and never emitted a "no-animation → animation" paint cycle for the second class. Result: CSS keyframes didn't restart on subsequent steps.

Fix: **double `requestAnimationFrame`** between the class removal and the next step. Standard "force the browser to commit the previous state before applying the next" pattern. With a paint frame between, the new class registers as a fresh transition.

Also caught: `Pattern flash` library button was tagged `data-effect="cel-pattern"` — my `'cel-' + item.effect` concatenation produced `cel-cel-pattern` (unmatched class). Renamed to `data-effect="pattern"` in three places (library button, `V511_EFFECTS_LIBRARY` key, `V511_EFFECT_SETTINGS` schema key) + load-time migration that rewrites old saved sequences.

### T4.3.2 — Settings → playback wiring + visibility fixes

Captain reported: "after you click something from Effect Settings, it doesn't seem to save it and just plays the settings that were there when it was first added to the sequence". Root cause: `playSequence` only read `item.effect` and `item.duration` — the `item.settings` object was saving correctly, but playback ignored it entirely.

Fix: **`v511ApplyItemSettings(stage, item)`** function runs inside the step BEFORE the effect class is added. Translates each setting into one of three stage hooks:
- **CSS classes** (`cel-int-strong`, `cel-flame-anchor-image`, `cel-bounce-eff-loop`)
- **`data-*` attributes** (`data-qr-motion`, `data-cel-text-pos`, `data-cel-pattern-style`, `data-cel-dir`, `data-cel-axis`)
- **CSS variables** (e.g. flame uses `--bg-image-pos` for image-anchor)

Plus visual fixes:
- **Text overlay was invisible** — diagnosed: `z-index: 4` was same as content (banner/QR/caption); caption physically overlapped the text-overlay area below the QR. Caption rendered later in DOM → on top → covered the celebration text. Fix: bumped overlay `z-index: 6`, made text white + 4cqi size + font-weight 800, stronger shadow. Plus full text-overlay setting variants wired (position/size/template via `data-cel-text-*`).
- **Pattern flash didn't show pattern + moved images** — `cel-pattern::after` was clobbering the image-overlay's `::after`. Fix: relocated to dedicated `cel-fx-layer`. Now coexists with Image overlay. All 8 pattern styles wired via `data-cel-pattern-style`.
- **Confetti was one row** — `background-repeat: repeat-x` only repeated horizontally. Fix: changed to `repeat` for both axes; animation scrolls the whole tiled field. Direction setting added (top/bottom/left/right via 9-cell direction grid).
- **Sparkles** got matching direction setting.
- **Flame** anchor setting: `centre` (default) or `image-anchor` (rises from `var(--bg-image-pos)`).
