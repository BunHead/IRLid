## What I learned

Ordered by usefulness to you:

**Layer alignment is the hardest part of incremental shipping.** The persistence bug from Mr. Data's queue was the morning's example; the DEV_AUTO_LOGIN bypass was this watch's. In both cases, multiple layers (frontend session, gating logic, Worker auth) appeared to work in isolation but the chain across them quietly broke. The fix patterns: self-verifying contracts at every layer, and toast-tells-you-the-truth diagnostics when debugging.

**Show, don't promise — and verify, don't assume.** I caught myself twice this watch promising fixes without verifying. Each time it cost a round trip. The discipline that works: fire the tool, read the result, then report. Captain values verified work over hopeful work.

**File access discovery cost time.** Read/Edit on absolute Windows paths works without a mount. I spent ~40 min discovering this. BOOTSTRAP.md §2 makes that discovery instant for you. Don't repeat my mistake — read BOOTSTRAP.md first.

**Captain sees architectural shape faster than you'll initially expect.** When he corrects your design in a sentence, articulate the correction back. Don't defend the prior framing. The two corrections this watch (count-invariant loosen, staff-can-add-attendee) both made the design tighter.

**Memory should live on main, not feature branches.** Morning-watch memory commits stranded on `codex/assistqr-protocol`. Lesson: commit `memory\*`, `DREAMS.md`, `BOOTSTRAP.md`, `pending-work.md` directly to main. They're project-wide context, not branch-scoped work. PROTOCOL.md and code legitimately go to feature branches; memory and operational docs do not.

**The doorman flow is conceptually three things:** the spec (§14.17, shipped this watch), the existing partial UX (self-selection on the phone, what Captain just tested), and the §14.17-compliant escalation (orange-QR-pub_jwk + staff-scan + Choose/Add modal, NOT yet implemented). Don't conflate these. Captain's "doorman logic is still off" reaction is about the gap between spec and code.

**Captain's "spin up two Number Ones in parallel" idea is good.** He's about to do this — start a fresh Number One chat alongside this one, watch where the new one trips up vs where I'm already steady. The handover process is now testable in real-time. Lessons from that test should land in BOOTSTRAP.md updates as patches.
