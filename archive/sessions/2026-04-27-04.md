**Live debugging — checkout QR bug:**
- Captain reported empty white box where inline checkout QR should render in `org.html` dashboard "Initiate check-out" action
- Number One identified root cause: `renderCheckoutQr()` calls `getElementById('checkoutQr_' + CSS.escape(checkinId))`. UUIDs starting with a digit (e.g. `9e179aa2-…`) get their leading "9" escaped to `\\39 ` by `CSS.escape`. But `getElementById` takes a literal string, not a CSS selector — so target is null, function early-returns, no QR rendered.
- Mr. Data's PR #44 ("checkout image fallback + checkout fullscreen logo fix") added an `<img>` API-backed fallback BUT kept the `CSS.escape` and the `if (!target) return` guard — so his fallback never fires either. PR #44 didn't fix the actual bug.
- Number One applied the one-line fix locally on test env working tree (drop `CSS.escape`)
