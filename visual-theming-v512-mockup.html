Design notes for Captain

  • Section 1 (Brand identity) is NEW — consolidates banner text + font + logo from scattered locations. Banner text template with {org} / {event} / {room} makes the "Test Event — Theme Demo" problem go away permanently.
  • Section 2 (Colour & background) folds the current Background + Pattern overlay + Image overlay expanders into one section. Sub-headers instead of three separate expanders.
  • Section 3 (QR appearance) is EXTRACTED from the current "QR customization" expander which bundled QR colour with celebration palette. QR is its own thing.
  • Section 4 (Celebrations) is the HEAVIEST section — all the creative-fun stuff. Celebration palette (separated from QR colour), sequence builder (relocated from "Accept behaviour" expander), text template (new from v5.11.21), sound. Mode picker stays at the top.
  • Section 5 (Post-accept flow) is EXTRACTED from the old "Accept behaviour" expander. Welcome message + redirect + dwell time are "what happens after" — they shouldn't live mixed with celebration effects. Smart receptionist toggle is future-work (v5.12+).
  • Org tab slimmed — logo upload + description tagline migrate to Visual Theming → Brand Identity. Org tab keeps: Display name, Slug, Website URL, Contact email, Org terms/disclaimer, Brand polish & contact info expander. Lean admin surface — set once, rarely revisit.
  • No new data shapes — the reorganisation is UI reshuffling only. All existing theme._v511.* persistence stays the same. The only new key is theme.bannerText (the template string, e.g. "{org} — {event}").
  • No Worker changes — same POST /org/settings round-trip.
