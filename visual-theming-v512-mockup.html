Anchor system redesign per Captain's 28 May feedback. Click any section header to expand. The Anchor System block inside Section 4 is the headline change. This file is a standalone demo — it does NOT touch the live site.

What's new in Rev 4

  • Anchor presets — Centre · Image · Logo · QR chips replace implicit/fixed anchor. Logo is just one option among four.
  • ⚓ Show crosshairs toggle — anchor-icon button toggles crosshair overlay on the preview. Visual affordance, not a text checkbox.
  • Multi-select mode — orange "Multi" toggle. When on, anchor chips become checkboxes and active anchors get A1 / A2 / A3 letter tags.
  • +-shape offset sliders — horizontal + vertical sliders intersecting visually as a +, sized to match the 9-cell direction grid. Drag H-slider left → anchor moves left. Slider body teaches function.
  • Per-effect anchor badge — every effect card in the sequence shows its current anchor at a glance (e.g. ⚓ Logo, ⚓ A1+A2).
  • Drag-to-anchor on preview — Photoshop-style: drag the crosshair onto Centre/Image/Logo/QR to snap-select.
  • Per-effect ↺ reset — restores Captain-recommended defaults for just that effect. Distinct glyph from refresh ↻.
  • Ctrl+Z / Ctrl+Y undo stack — global undo across the panel. Pill at the top of Section 4 indicates state.
  • Tooltips on anchor names — explain Image/Logo/QR spatially. aria-describedby for WCAG 2.1 SC 3.3.2 compliance.

Polish carry-overs (Captain's 28 May review)

  • Ripple effect — border around frame looks odd (current v5.11.24a). Effect's outer radial gradient may be painting past the orange-dome rounded bottom corners. Either clip to stage shape with border-radius: inherit on the effect layer, or constrain the radial gradient size to fit within the stage's inscribed rectangle. Fixable as a small v5.11.24b inline patch OR rolled into v5.12.0.
  • Vignette flash — same border-around-frame issue. Vignette uses a radial gradient from edges-inward; if the stage has rounded corners the corners can paint through. Same clip-to-stage fix.
  • EDITING: DENY pill — magenta border framing the live preview when in Deny editing mode might distract from preview content. Worth deciding: keep as honest "you are editing X mode" signal, or move to a less prominent badge above the preview frame.
  • ↺ Reset glyph — must be visually distinct from refresh ↻ to avoid confusion. Candidates: ↶ (rotate anti-clockwise with serif tail), ⟲ (reload with arrow), plain word "Default", or ↩ (return arrow). Decision: pick at Rev 5 review.
  • Inline-vs-fullscreen offset dialling — solved architecturally by storing anchor positions as percentage offsets from the anchor target's centre, not absolute pixels. Same percentage values render correctly at any stage size — inline (400px), fullscreen (1920px), 4a phone (720px) — because everything scales proportionally to the target element's bounding box. Drag-to-anchor UI updates the percentage; render code reads same percentage and computes pixels per stage.
  • W3C accessibility — tooltips on anchor names use aria-describedby pattern pointing to a visually-hidden span. WCAG 2.1 SC 3.3.2 (Labels or Instructions) and 1.3.1 (Info and Relationships). Same anchor name always means the same spatial position across the whole app: SC 3.2.4 (Consistent Identification). Real boost for the W3C cert ambition.
  • Ctrl+Z / Ctrl+Y undo stack — global undo across the whole panel. 50-entry ring buffer of theme-state snapshots; every meaningful change pushes a new state. Per-effect ↺ Reset is a narrower scope: restores Captain-recommended defaults for that effect only.

Design notes for Captain

  • Section 4 is the headline of Rev 4. The Anchor System block sits between Effect Sequence and Text Template. It only configures the currently-selected effect (blue-outlined card in the sequence). Selecting a different effect card swaps the anchor block's contents.
  • Anchor controls are per-effect, not global. Different effects need different anchors (Glow rays = Logo, Ripple = Centre, Stream = QR). The ⚓ "Show crosshairs" toggle is the only global control — it just decides whether the crosshair overlay is visible on the preview frame.
  • Multi-select adds the A1/A2 letter tags. Single-select mode = one anchor, named by its target ("Logo"). Multi-select mode = multiple anchors, labelled by order added (A1, A2, A3). Effect cards show ⚓ A1+A2 when multi is active for that effect.
  • + slider layout teaches the function. Horizontal slider running L↔R lives on the X axis. Vertical slider running ↕ lives on the Y axis. They cross visually at the anchor centre. Cognitively: dragging H-slider physically left moves the anchor left. No translation step.
  • Sensible defaults per effect type. Confetti = full-screen (no anchor — explicit "— full-screen —" badge on card). Ripple = Centre. Sparkles = Image. Stream = QR. Glow rays = Logo. Apes can change but the first run looks right without thinking.
  • No new data shapes beyond Rev 2. Anchor state lives inside each effect's existing settings object as anchor: { targets: ['logo'], offset: { x: -10, y: 5 }, direction: 'centre' }. Backward compat: missing anchor field falls back to effect's default (the "sensible default").
  • No Worker changes. Same POST /org/settings round-trip.
